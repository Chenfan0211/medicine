## 1. 系统概述

### 1.1 产品定位

智合医药 ERP + 中台（一期）聚焦合同约定的四项核心能力：

1. **采购对账自动化（M1）** — 合同→采购→入库→发票→三单比对→对账单→供应商确认→返利
2. **门店补货计划（M2）** — 销量归集→预测→安全库存→补货建议→供应商匹配→转采购
3. **三平台商品库存（M3）** — 主数据→一键上架/定价→库存中心→防超卖→订单回传→履约出库
4. **基础平台（M4）** — 组织权限、看板、通知、日志、备份

原型中「二期·扩展功能」菜单（会员、处方、配送、平台财务账单等）为历史原型保留，**不在一期合同 62 项验收范围内**。

### 1.2 用户角色（建议）

| 角色 | 典型权限范围 |
|------|-------------|
| 超级管理员 | 系统配置、用户角色、全模块 |
| 采购员 | M1 采购单、入库、发票、三单复核（黄） |
| 采购主管 | 采购审批（5–20万）、三单复核（红） |
| 财务 | 采购审批（>20万）、对账、返利 |
| 运营人员 | M2 补货建议审核、M3 商品/平台同步 |
| 门店店长 | 本店库存、订单履约、数据权限内门店 |
| 供应商（门户） | M1-4 对账单查看/确认/反馈 |

> 原型中权限通过「角色管理 → 权限」及「菜单管理 → 权限标识」配置，权限标识数据来源于字典 `sys_perm`。

### 1.3 全局 UI 约定

| 区域 | 用法 |
|------|------|
| **面包屑** | 显示当前模块路径，如「订单管理 / 波次管理」 |
| **搜索区** | 填写条件后点「查询」过滤列表；下拉选「全部」表示不过滤 |
| **工具栏** | 页面级批量操作（新增、导出、同步等） |
| **表格行操作** | 每行右侧「编辑/详情/删除」等，点击弹出抽屉或确认框 |
| **抽屉（Drawer）** | 新增/编辑/请货等表单，填写后点「保存」 |
| **详情抽屉** | 只读展示字段，不可编辑 |
| **确认框** | 删除、回库、取消等不可逆操作需二次确认 |
| **分页** | 每页 20 条，支持上一页/下一页/页码切换 |
| **消息提示** | 操作成功/失败以 Toast 形式反馈 |
| **操作日志** | 关键业务操作写入「系统管理 → 操作日志」 |

### 6.6 拣货出库

**路径**：订单管理 / 拣货出库

> 对应流程图：请货 → 拣货 → 生成快递单 → 打包 → 打印面单 → 出库

#### 6.6.1 查询条件

- 订单号
- 状态：全部 / 未请货 / 已请货 / 已拣货 / 已出库 / 异常 / 已回库

#### 6.6.2 工具栏

| 按钮 | 用法 |
|------|------|
| **批量请货** | 对所有「未请货」明细按默认请货数量批量请货 |
| **管理发货地址** | 跳转「发货地址」页维护发件人信息 |

#### 6.6.3 行操作与状态机

```
未请货 ──请货──► 已请货 ──确认拣货──► 已拣货 ──生成快递单──► (有单号)
   │                │                    │
   │                │                    ├──打包──► (提示完成)
   │                │                    ├──打印面单──► (提示完成，需先有快递单)
   │                │                    └──出库──► 已出库
   └──回库──► 已回库          异常 ──回库──► 已回库
```

| 操作 | 当前状态 | 用法详解 |
|------|---------|---------|
| **请货** | 未请货 | 弹出抽屉，可修改「请货数量」（默认取订单数量）；提交后调用 ERP 请货接口；库存不足则标记「异常」 |
| **确认拣货** | 已请货 | ERP 拣货成功后执行；扣减可用库存，写出库流水；明细 →「已拣货」 |
| **回库** | 未请货 / 已请货 / 异常 | 二次确认；若已拣货则库存回增；明细 →「已回库」 |
| **生成快递单** | 已拣货 | 生成快递单号（如 SF 开头）；需先维护发货地址 |
| **打包** | 已拣货 | 需已生成快递单；标记打包完成 |
| **打印面单** | 已拣货 | 需已生成快递单；触发面单打印 |
| **出库** | 已拣货 | 需已生成快递单；明细 →「已出库」，关联订单 →「配送中」；**定时任务回传 ERP 生成出库单** |
| **详情** | 任意 | 查看波次号、订单号、商品、请货数、拣货数、快递单号、状态 |

**请货业务规则**：

- 请货数量可手动修改后再提交
- 若门店该 SKU 可用库存 < 请货数量 → 请货失败，状态变「异常」
- 请货成功后波次状态同步更新为「已请货」

**回库场景**（流程图说明）：

- 订单已分配库存但用户取消或订单异常时，执行回库释放库存

## 15. 渠道订单履约 — 端到端操作手册

以下为一线操作人员标准作业流程（SOP）：

### 步骤 1：拉取订单

1. 进入 **渠道订单池** 或 **履约流程**
2. 点击「拉取美团」「拉取饿了么」「拉取京东」或「拉取三渠道订单」
3. 确认列表出现新订单，履约状态为「已拉取」

### 步骤 2：锁单（可选）

- 暂不发货的订单 → 行操作「锁单」
- 恢复发货 →「解锁」

### 步骤 3：合单（可选）

- 存在相同手机+地址+商品的多单时 →「自动合单」
- 在 **合单管理** 查看合单组

### 步骤 4：配置波次策略（首次/调整时）

- 进入 **波次策略** →「+ 新增策略」
- 设置规则说明、每批上限、波次类型

### 步骤 5：生成波次

- **普通订单**：渠道订单池或波次管理 →「生成波次」
- **合单订单**：合单管理 →「合单波次生成」
- 在 **波次管理** Tab「未请货」查看新波次

### 步骤 6：请货

- **波次管理** → 行操作「请货」（整波次批量）
- 或 **拣货出库** → 行操作「请货」（单笔，可改数量）

### 步骤 7：确认拣货

- ERP 回传拣货成功后 → **拣货出库** →「确认拣货」
- 库存扣减，状态 →「已拣货」

### 步骤 8：异常回库（如需要）

- 订单取消/异常 →「回库」→ 库存释放

### 步骤 9：维护发货地址（首次/变更时）

- **发货地址** →「+ 新增地址」→ 设默认

### 步骤 10：发货出库

1. 「生成快递单」→ 获得快递单号
2. 「打包」
3. 「打印面单」
4. 「出库」→ 订单进入配送，ERP 出库单由定时任务回传
